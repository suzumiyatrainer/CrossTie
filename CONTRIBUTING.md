# Contributing Guide

このプロジェクトへの貢献をいただきありがとうございます。

## セキュリティに関する注意

### 秘密情報の取り扱い

**絶対にコミットしないでください：**
- API キー、アクセストークン
- パスワード、秘密鍵
- データベース認証情報
- AWS/Azure/GCP 認証トークン
- その他の認証情報

秘密情報が必要な場合は：
1. `.env.example` に構造を示す
2. `.env` または環境変数で管理
3. `.gitignore` に追加

## Pull Request プロセス

1. **フォークしてフィーチャーブランチを作成**
   ```bash
   git checkout -b feature/your-feature
   ```

2. **コミットメッセージの規則**
   ```
   feat: 新機能の説明
   fix: バグ修正の説明
   docs: ドキュメント更新
   style: コード整形
   refactor: コードの再構成
   perf: パフォーマンス改善
   test: テストの追加
   chore: ビルド/依存関係の変更
   ```

3. **セキュリティチェック**
   - 秘密情報が含まれていないか確認
   - 依存関係に脆弱性がないか確認
   - コード品質チェックに合格

4. **Pull Request を作成**
   - 説明的なタイトルを記入
   - 変更内容の詳細を説明
   - 関連する Issue をリンク

5. **レビューと承認**
   - 最低1人のレビューが必須
   - すべてのセキュリティチェックに合格
   - マージ可能な状態

## セキュリティレポート

脆弱性を発見した場合は、[SECURITY.md](./SECURITY.md)の手順に従い、公開せずに報告してください。

## コード品質基準

- Java コードは Google Java Style Guide に準拠
- テストカバレッジ最小70%
- すべてのテストが成功
- セキュリティスキャンに合格
